# 구글 애드센스 설정 가이드 (Share Toilet)

애드센스 계정이 변경(`pub-8142649369272916`)됨에 따라, 우리 앱(대똥단결)에 광고를 제대로 나오게 하기 위한 설정 방법을 안내해 드립니다.

## 1. 개요 (중요)

보내주신 스크린샷 화면("사이트 기준 자동 광고 설정")은 구글이 **알아서** 빈 공간에 광고를 넣는 기능입니다.
하지만 우리는 이미 **개발자가 수동으로 최적의 위치(목록 사이, 모달 하단 등)에 광고 코드를 심어둔 상태**입니다.

따라서 두 가지 전략 중 하나를 선택해야 합니다:

### 추천 전략: "수동 광고 단위" 사용 (UI 깨짐 방지)
우리가 정성스럽게 배치한 위치에만 깔끔하게 나오게 하는 방법입니다. 지도나 버튼을 가리는 '자동 광고'의 부작용을 막을 수 있습니다.

---

## 2. 따라해주셔야 할 것 (Step-by-Step)

### 단계 1: 광고 단위(Ad Unit) 만들기
애드센스 관리자 페이지에서 **[광고] > [광고 단위 기준]** 메뉴로 이동합니다.

다음 3가지 종류의 광고 단위를 각각 새로 생성해주세요.

1.  **디스플레이 광고 (Display ads)** - *가장 많이 씀*
    *   이름: `Mypage_Banner` 등 식별하기 쉬운 이름
    *   형식: 사각형 (Square) 또는 수평형 (Horizontal) -> *반응형(Responsive)* 선택
    *   **만들기** 클릭 후, 코드에서 **`data-ad-slot="1234567890"`** 숫자(Slot ID)를 복사해둡니다.

2.  **네이티브 광고 (Native Multiplex ads)** - *목록용 (선택사항)*
    *   우리는 현재 '디스플레이 광고'를 네이티브처럼 쓰고 있으므로, 위 1번(디스플레이)만 만드셔도 충분합니다.

### 단계 2: ID 입력하기
새로 발급받은 **Slot ID(숫자)**들을 개발자에게 알려주시거나, 코드의 `src/constants/adUnits.ts` 파일에 직접 입력합니다.

#### 📋 [추천] 생성해야 할 광고 단위 목록
우리 앱의 각 위치에 최적화된 광고 단위를 각각 만드시는 것을 강력 추천합니다. 이름은 예시처럼 지으시면 관리하기 편합니다.
(모두 **디스플레이 광고 - 반응형**으로 만드시면 됩니다)

| 위치 설명 | 추천 이름 (예시) | 코드 상 변수명 (참고용) |
| :--- | :--- | :--- |
| **메인 홈 목록 상단** | `Home_List_Top` | `HOME_LIST_TOP` |
| **메인 홈 목록 중간** (피드) | `Home_List_Feed` | `HOME_LIST_INTERLEAVED` |
| **메인 홈 하단 배너** | `Home_Bottom_Banner` | `HOME_BOTTOM_FAB` |
| **화장실 상세 - 리뷰 목록** | `Detail_Review_List` | `DETAIL_REVIEW_NATIVE` |
| **화장실 상세 - 지도 하단** | `Detail_Map_Bottom` | `DETAIL_MAP_BOTTOM` |
| **화장실 제보 - 폼 중간** | `Submit_Form_Mid` | `SUBMIT_FORM_MIDDLE` |
| **마이페이지 - 중간** | `MyPage_Mid` | `MYPAGE_MID_BANNER` |
| **마이페이지 - 하단** | `MyPage_Bottom` | `MYPAGE_BOTTOM_BANNER` |
| **각종 모달(팝업) 하단** | `Modal_Bottom_Common` | `MODAL_...` (공통 사용 가능) |

> **꿀팁**: 너무 많아서 귀찮으시다면, **`App_Common_Display`** 같이 하나만 만드시고 그 **ID 하나를 모든 곳에 복사**해서 넣으셔도 작동은 합니다. (하지만 수익 분석이 어렵습니다.)

```typescript
// src/constants/adUnits.ts 예시
export const AD_UNITS = {
    HOME_LIST_TOP: "1234567890",       // <- Home_List_Top의 ID
    DETAIL_PASSWORD_BOTTOM: "0987654321", // <- Detail_Review_List의 ID
    ...
};
```

### 단계 3: 사이트 기준 설정 (질문하신 화면)
**[광고] > [사이트 기준] > [수정(연필 아이콘)]** 으로 들어갑니다.

1.  **자동 광고(Auto Ads)**: **[끄기(OFF)]** 를 권장합니다.
    *   *이유*: 켜두면 지도 위나 버튼 위에 뜬금없이 광고가 덮여서 앱 사용이 불편해질 수 있습니다.
2.  **앵커 광고(Anchor ads)**: (선택) 켜셔도 됩니다. 화면 맨 하단/상단에 떠있는 광고입니다. (우리 앱엔 이미 수동 하단 배너가 있어서 겹칠 수 있으니 끄는 게 깔끔합니다.)
3.  **전면 광고(Vignette ads)**: (선택) 페이지 이동 시 화면 전체를 덮는 광고입니다. 수익은 좋지만 사용자 경험엔 안 좋습니다.

**결론**: 우리 앱에는 이미 '수동 광고'가 촘촘히 배치되어 있으므로, **자동 광고 설정은 건드리지 않거나 끄시는 것**이 가장 좋습니다.

---

## 3. 요약

1.  **자동 광고 설정 화면**에서는 아무것도 안 켜셔도 됩니다(또는 다 끄세요).
2.  대신 **[광고 단위 기준]** 탭에서 '디스플레이 광고'를 여러 개 생성해서, 그 **ID 숫자들**을 모아주시는 게 핵심입니다.
3.  그 ID들을 코드에 넣으면 배포 시점에 바로 적용됩니다.
